# Membership Card (Tessera) Feature

## Overview

When a user subscribes to Ekidna APS through the website, they automatically receive a personalized membership card (tessera) as a PDF attachment via email.

## Features

- **Automatic Generation**: Membership cards are automatically generated when a subscription is created
- **Unique Serial Numbers**: Each subscriber receives a unique, sequential serial number (starting from 1)
- **Personalized**: Cards include the subscriber's name, subscription year, and serial number
- **Email Delivery**: Cards are attached to the confirmation email sent to subscribers
- **Professional Formatting**: Text is rendered in italic/cursive style with black color

## Membership Card Content

The card includes two main sections:

### Top Section (Smaller)
Contains the subscription year and serial number
- Format: `2025 - N. 00123`
- Font: Helvetica Oblique (italic), 18pt, black

### Bottom Section (Larger)
Contains the subscriber's full name
- Format: `NOME COGNOME` (uppercase)
- Font: Helvetica Oblique (italic), 24pt, black

## Database Schema

### New Field: `serial_number`

```sql
ALTER TABLE subscribers ADD COLUMN serial_number INTEGER UNIQUE NOT NULL;
```

- **Type**: Integer
- **Unique**: Yes (each subscriber gets a unique number)
- **Auto-generated**: Yes (via PostgreSQL sequence)
- **Sequence**: `subscribers_serial_seq`

## How It Works

### Subscription Flow

1. **User submits subscription form** (website)
   ↓
2. **Backend creates subscriber record**
   - Database auto-generates `serial_number` via sequence
   - Record saved with all subscriber data
   ↓
3. **PDF Service generates membership card**
   - Loads template PDF from `/pdf/tessera_ekidna.pdf`
   - Adds year and serial number to top zone
   - Adds subscriber name to bottom zone
   - Returns PDF as Buffer
   ↓
4. **Email Service sends confirmation email**
   - Attaches generated PDF
   - Includes serial number in email body
   - Mentions attached membership card
   ↓
5. **Subscriber receives email with card**

### Serial Number Generation

Serial numbers are automatically generated by PostgreSQL using a sequence:

```sql
CREATE SEQUENCE subscribers_serial_seq START 1;

-- Auto-assigned on INSERT
INSERT INTO subscribers (...)
VALUES (...)
RETURNING serial_number;  -- Returns: 1, 2, 3, etc.
```

## PDF Generation Service

### `pdfService.generateMembershipCard(data)`

Generates a personalized membership card PDF.

**Parameters:**
```javascript
{
  name: string,        // Subscriber's full name
  serialNumber: number, // Sequential number (from DB)
  year: number         // Subscription year
}
```

**Returns:**
- `Promise<Buffer>` - PDF file as buffer

**Example:**
```javascript
const pdfService = require('./services/pdfService');

const pdfBuffer = await pdfService.generateMembershipCard({
  name: 'Marco Rossi',
  serialNumber: 123,
  year: 2025
});
```

## Email Service Updates

### `emailService.sendSubscriptionConfirmation(subscriber, attachments)`

Now accepts an optional `attachments` parameter.

**Attachments Format:**
```javascript
[{
  filename: 'Tessera_Ekidna_2025_00123.pdf',
  content: Buffer,
  contentType: 'application/pdf'
}]
```

**Supported Transports:**
- **Brevo HTTP API**: Attachments are base64-encoded automatically
- **SMTP (Nodemailer)**: Attachments are passed as Buffer objects

## Troubleshooting

### Migration Issues

**Problem**: Migration fails with "column already exists"
```
ERROR: column "serial_number" already exists
```

**Solution**: The field already exists (possibly from running init.sql). Skip migration.

---

**Problem**: Migration fails with "sequence already exists"
```
NOTICE: relation "subscribers_serial_seq" already exists
```

**Solution**: This is just a notice, not an error. Migration succeeded.

### PDF Generation Issues

**Problem**: "Template file not found"
```
Error: ENOENT: no such file or directory
```

**Solution**: Ensure `/pdf/tessera_ekidna.pdf` exists:
```bash
ls -l pdf/tessera_ekidna.pdf
```

---

**Problem**: Text not visible on PDF

**Solution**:
- Check PDF template has white/light colored blank zones
- Verify text coordinates in `pdfService.js` (topY, bottomY)
- Adjust font size if text is too large

### Email Attachment Issues

**Problem**: Email sent but no attachment

**Solution**: Check logs for PDF generation errors:
```bash
# Look for these messages in logs:
# "🎫 Generating membership card for: [name]"
# "✅ Membership card generated successfully"
# "📎 Attachments: Tessera_Ekidna_..."
```

---

**Problem**: Brevo API attachment error

**Solution**: Ensure attachments are base64-encoded. The `emailService` handles this automatically.

## Customization

### Changing Font

To use a custom cursive font instead of Helvetica Oblique:

1. Add TTF font file to `backend/fonts/`
2. Update `pdfService.js`:

```javascript
const fs = require('fs').promises;

// Load custom font
const fontBytes = await fs.readFile('fonts/YourCursiveFont.ttf');
const customFont = await pdfDoc.embedFont(fontBytes);

// Use custom font
firstPage.drawText(text, {
  font: customFont,
  // ... other options
});
```

### Adjusting Text Position

Edit coordinates in `pdfService.js`:

```javascript
// Top zone position
const topY = height - 155; // Increase to move up, decrease to move down

// Bottom zone position
const bottomY = height - 630; // Increase to move up, decrease to move down
```

### Changing Text Format

Modify formatting in `pdfService.js`:

```javascript
// Top text format
const topText = `${year} - N. ${String(serialNumber).padStart(5, '0')}`;
const topFontSize = 18;

// Bottom text format
const bottomText = name.toUpperCase(); // Currently uppercase
const bottomFontSize = 24;
```
